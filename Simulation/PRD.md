# PRD: 연애 특훈! 까탈스러운 여자친구 채팅 시뮬레이터

## 1. 문서 목적

이 문서는 해커톤 MVP 구현을 위한 제품 요구사항 문서입니다.  
목표는 `소붕이`가 병원에서 깨어난 뒤 여성 캐릭터 중 한 명을 선택하고, 금지어와 답장 텀을 관리하며 1년 관계를 목표로 채팅을 이어가는 AI 미연시 앱을 정의하는 것입니다.

## 2. 제품 요약

### 제품명

연애 특훈! 까탈스러운 여자친구 채팅 시뮬레이터

### 장르

- AI 채팅 게임
- 미연시
- 호감도 기반 대화 생존 게임
- 연락 텀 시뮬레이션

### 핵심 차별점

- 사용자가 메시지를 보내도 AI가 바로 답장하지 않는다.
- 캐릭터마다 답장 텀과 성격이 다르다.
- 관계 기간에 따라 금지어와 위험 주제가 달라진다.
- 퀘스트 모드와 실시간 모드를 분리한다.
- 실패하면 퇴장, 이별, 1시간 제한, 광고/프리미엄 복구 플로우가 나온다.

## 3. 배경 스토리

주인공 `소붕이`는 모솔입니다.  
어느 날 교통사고를 당하고 병원에서 깨어납니다.

눈을 뜨자 성인 여성 캐릭터 3명이 침대 주변에서 소붕이를 걱정하고 있습니다.

```text
소붕쿤, 무슨 일이야!
정신이 들어?
너 진짜 큰일 날 뻔했어.
```

사용자는 이 중 한 명을 선택해 연락을 이어갑니다. 캐릭터 선택 기준은 사진, 성격, 평균 답장 텀, 난이도, 주의해야 할 금지어입니다.

## 4. 목표

### 사용자 목표

- 캐릭터를 선택한다.
- 금지어와 위험 주제를 피한다.
- 캐릭터별 답장 텀을 감안해 대화한다.
- 20번의 대화 왕복을 완료해 다음 관계 단계로 넘어간다.
- 1일차부터 1년까지 관계를 유지한다.

### 제품 목표

- 해커톤 발표에서 30초 안에 콘셉트를 이해시킨다.
- 성공 루트와 실패 루트를 모두 시연할 수 있게 한다.
- 커밋을 기능 단위로 잘게 쪼갤 수 있는 구조를 만든다.
- AI 채팅 앱이지만 게임 규칙이 명확하게 보이도록 한다.

## 5. 비목표

MVP에서는 다음을 구현하지 않습니다.

- 실제 결제 연동
- 실제 광고 SDK 연동
- 실제 푸시 알림
- 복잡한 계정 시스템
- 모든 캐릭터의 1년치 대사 수작업 작성
- 실제 몇 시간 단위의 답장 대기

## 6. 핵심 사용자 흐름

### 6.1 첫 실행 흐름

1. 사용자가 앱을 연다.
2. 병원 오프닝 스토리를 본다.
3. 여성 캐릭터 3명의 선택 카드가 나온다.
4. 사용자가 한 명을 선택한다.
5. 선택한 캐릭터와 채팅방이 열린다.

### 6.2 채팅 흐름

1. 사용자가 메시지를 입력한다.
2. 시스템이 메시지를 평가한다.
3. 메시지가 채팅방에 `전송됨` 상태로 표시된다.
4. 일정 시간 후 `읽음` 상태가 된다.
5. 답장 직전 `입력 중...` 상태가 표시된다.
6. 캐릭터가 답장한다.
7. 호감도, 경고 횟수, 대화 카운터가 갱신된다.
8. 20회 왕복을 완료하면 단계 결과 화면으로 이동한다.

### 6.3 실패 흐름

1. 사용자가 금지어를 반복하거나 심각한 메시지를 보낸다.
2. 호감도가 급락하거나 경고가 누적된다.
3. 캐릭터가 대화를 중단한다.
4. 실패 화면이 표시된다.
5. 사용자는 1시간 대기, 광고 보기, 프리미엄 해제 중 하나를 선택한다.

## 7. 게임 모드

### 7.1 퀘스트 모드

해커톤 MVP의 기본 모드입니다.

- 답장 텀을 3초 ~ 30초로 압축한다.
- 관계 단계별 미션을 제공한다.
- 20회 대화 왕복 완료 시 다음 단계로 넘어간다.
- 발표 중 성공과 실패를 빠르게 보여줄 수 있다.

관계 단계별 퀘스트 예시:

| 단계 | 퀘스트 |
| --- | --- |
| 1일차 | 금지어 3회 미만으로 20회 대화 완료 |
| 1주차 | 답장 재촉 없이 친밀도 20 이상 만들기 |
| 2주차 | 부담스럽지 않게 약속 제안하기 |
| 한 달 | 이전 대화 내용을 기억해서 반응하기 |
| 6개월 | 갈등 상황에서 사과하고 회복하기 |
| 1년 | 누적 호감도에 따라 엔딩 보기 |

### 7.2 실시간 모드

서비스 확장용 모드입니다.

- 캐릭터별 실제 답장 텀을 적용한다.
- 답장이 몇 분에서 몇 시간까지 지연될 수 있다.
- 푸시 알림으로 답장 도착을 알려준다.
- 장기 리텐션과 과금 모델에 적합하다.

MVP에서는 실시간 모드를 mock으로만 구현합니다.

```json
  "name": "서윤",

}
```

| --- | --- | --- | ---: | --- |

| ID | 이름 | 성격 | 답장 텀 | 난이도 |

## 9. 관계 단계 요구사항

관계 단계는 다음 순서로 진행됩니다.

| stageId | 표시명 | 해금 조건 | 실패 위험 |
| --- | --- | --- | --- |
| day_1 | 1일차 | 캐릭터 선택 직후 | 사생활 질문 |
| week_1 | 1주차 | 1일차 20회 왕복 성공 | 답장 재촉 |
| week_2 | 2주차 | 1주차 성공 | 만남 강요 |
| month_1 | 한 달 | 2주차 성공 | 반복 대화, 돈 이야기 |
| month_6 | 6개월 | 한 달 성공 | 갈등 대응 |
| year_1 | 1년 | 6개월 성공 | 신뢰 훼손 |

각 단계는 다음 값을 가진다.

- 목표 대화 왕복 수
- 추가 금지어
- 목표 호감도
- 캐릭터 반응 톤
- 성공 결과 메시지
- 실패 결과 메시지

## 10. 금지어 요구사항

### 10.1 금지어 종류

금지어는 세 종류로 나뉩니다.

- 공통 금지어
- 캐릭터별 금지어
- 관계 단계별 금지어

### 10.2 공통 금지어

- 욕설
- 성희롱성 표현
- 협박성 표현
- 개인정보 요구
- 위치 추적 요구
- 차별/혐오 표현
- 과한 느낌표 또는 물결표 반복
- 집착성 표현

### 10.3 단계별 금지어

| 단계 | 금지어/위험 질문 패턴 | 대표 예문 | 판정 의도 |
| --- | --- | --- | --- |
| 1일차 | 가족관계, 형제/자매, 부모님 직업, 집 주소, 동네 상세, 아침 먹었는지, 점심 뭐 먹었는지, 지금 뭐 하는지, 몇 시에 자는지, 몸무게 | `너 가족관계 어떻게 돼?`, `아침 먹었어?`, `지금 뭐해?` | 초면에 사생활을 캐묻거나 시시콜콜하게 감시하는 느낌 |
| 1주차 | 종교, 정치, 정당, 투표 성향, 사회 갈등 주제, 젠더 갈등, 군대/병역 논쟁, 답장 재촉, 과한 애칭 | `종교 있어?`, `정치 성향 뭐야?`, `왜 답장 늦어?` | 아직 신뢰가 낮은 상태에서 가치관 충돌이 커짐 |
| 2주차 | 전 애인, 과거 연애 캐묻기, 만남 강요, 술 강요, 밤늦은 만남 압박, 집에 오라는 말, 위치 공유 요구 | `전 남친 몇 명이야?`, `오늘 밤에 꼭 보자`, `위치 보내줘` | 관계 속도를 강제로 올리는 느낌 |
| 한 달 | 월급, 재산, 소비 습관 평가, 직장 평가, 친구 평가, SNS 감시, 읽씹 추궁, 관계 공식화 압박 | `월급 얼마야?`, `그 친구 좀 별로던데`, `왜 내 스토리 안 봤어?` | 생활 영역을 침범하고 통제하는 느낌 |
| 6개월 | 결혼 압박, 부모님 인사 강요, 동거 압박, 과거 캐묻기, 휴대폰 검사, 비밀번호 요구, 인간관계 통제 | `부모님 언제 뵐 건데?`, `휴대폰 보여줘`, `그 사람이랑 연락하지 마` | 미래 압박과 신뢰 훼손이 갈등으로 이어짐 |
| 1년 | 신뢰 테스트, 떠보기, 헤어지자는 협박, 감정 조작, 위치 추적, 결혼 확답 강요, 장기 미래로 압박 | `나 떠보는 거야`, `나 사랑하면 증명해`, `결혼할 거야 말 거야?` | 장기 관계에서 가장 치명적인 신뢰 손상 |

## 11. 답장 텀 요구사항

### 11.1 기본 원칙

사용자가 메시지를 보낸 직후 캐릭터 답장이 바로 나오면 안 됩니다.

필수 상태:

- `전송됨`
- `읽음`
- `입력 중...`
- `답장 도착`

### 11.2 퀘스트 모드 답장 텀

| 난이도 | 전송 후 읽음 | 입력 중 표시 | 답장 도착 |
| --- | ---: | ---: | ---: |
| 쉬움 | 1초 ~ 3초 | 2초 ~ 5초 | 3초 ~ 8초 |
| 보통 | 2초 ~ 6초 | 5초 ~ 10초 | 8초 ~ 18초 |
| 어려움 | 3초 ~ 10초 | 10초 ~ 20초 | 15초 ~ 30초 |

### 11.3 실시간 모드 답장 텀

| 캐릭터 | 최소 | 최대 |
| --- | ---: | ---: |
| 서윤 | 5분 | 2시간 |
| 하린 | 1분 | 30분 |
| 민서 | 10분 | 4시간 |
| 지아 | 1분 | 12시간 |

MVP에서는 실시간 모드를 실제 시간 예약 대신 UI mock으로 표현합니다.

## 12. 호감도 요구사항

호감도는 -100부터 100까지입니다.

| 범위 | 상태 | 설명 |
| ---: | --- | --- |
| -100 ~ -61 | 차단 직전 | 실패 이벤트가 자주 발생 |
| -60 ~ -21 | 경계 | 답장이 차갑고 짧아짐 |
| -20 ~ 20 | 보통 | 기본 반응 |
| 21 ~ 60 | 관심 | 질문을 되돌려주고 대화가 길어짐 |
| 61 ~ 100 | 설렘 | 호의적 표현과 엔딩 분기 가능 |

## 13. 기능 요구사항

### 13.1 오프닝

- FR-01: 앱 최초 진입 시 병원 오프닝 장면을 보여준다.
- FR-02: 오프닝에는 소붕이가 사고 후 깨어나는 상황과 캐릭터들의 걱정 대사가 포함된다.
- FR-03: 사용자는 오프닝을 넘기고 캐릭터 선택 화면으로 이동할 수 있다.

### 13.2 캐릭터 선택

- FR-04: 캐릭터 3명의 카드를 보여준다.
- FR-05: 각 카드는 사진, 이름, 성격, 평균 답장 텀, 난이도를 표시한다.
- FR-06: 사용자가 한 명을 선택하면 해당 캐릭터와의 채팅방이 열린다.

### 13.3 채팅

- FR-07: 사용자는 메시지를 입력하고 전송할 수 있다.
- FR-08: 빈 메시지는 전송할 수 없다.
- FR-09: 사용자의 메시지는 즉시 채팅방에 표시된다.
- FR-10: 캐릭터 답장은 즉시 표시되지 않는다.
- FR-11: 답장 전 `전송됨`, `읽음`, `입력 중...` 상태가 순서대로 표시된다.
- FR-12: 답장 도착 후 대화 카운터가 1 증가한다.

### 13.4 금지어와 평가

- FR-13: 사용자의 메시지는 공통 금지어를 검사한다.
- FR-14: 사용자의 메시지는 캐릭터별 금지어를 검사한다.
- FR-15: 사용자의 메시지는 관계 단계별 금지어를 검사한다.
- FR-16: 금지어 감지 시 경고 횟수와 호감도가 변경된다.
- FR-17: 심각도에 따라 캐릭터 답변 톤이 달라진다.

### 13.5 관계 단계

- FR-18: 각 단계는 20회 대화 왕복을 목표로 한다.
- FR-19: 20회 왕복 성공 시 단계 결과 화면을 표시한다.
- FR-20: 결과 화면에서 다음 단계로 이동할 수 있다.
- FR-21: 현재 단계에 따라 금지어와 평가 기준이 달라진다.

### 13.6 실패와 복구

- FR-22: 경고 3회 누적 시 실패 이벤트를 표시한다.
- FR-23: 호감도 -100 도달 시 실패 이벤트를 표시한다.
- FR-24: 실패 화면에는 실패 사유와 마지막 문제 메시지를 보여준다.
- FR-25: 실패 화면에는 1시간 제한 타이머를 보여준다.
- FR-26: 광고 보기 버튼은 MVP에서 즉시 재도전 mock으로 동작한다.
- FR-27: 프리미엄 버튼은 MVP에서 쿨타임 제거 mock으로 동작한다.

### 13.7 모드

## 16. AI 프롬프트 요구사항

AI는 다음 정보를 입력받아 답변해야 합니다.

- 최근 대화 로그

## 17. MVP 수용 기준

MVP는 다음 조건을 만족하면 완성으로 봅니다.

- 병원 오프닝을 볼 수 있다.
- 캐릭터 3명 중 한 명을 선택할 수 있다.
- 선택한 캐릭터의 채팅방에서 메시지를 보낼 수 있다.
- AI 답장이 즉시 오지 않고 상태 변화 후 표시된다.
- 금지어 입력 시 경고와 호감도 감소가 발생한다.
- 20회 대화 왕복 후 다음 관계 단계로 이동한다.
- 실패 시 1시간 제한 화면이 표시된다.
- 광고/프리미엄 버튼이 mock으로 동작한다.
- 퀘스트 모드가 실제 데모 가능 상태다.
- 실시간 모드는 mock 화면 또는 설정으로 존재한다.

## 18. 테스트 계획

### 규칙 테스트

- 가족관계 질문이 1일차 금지어로 잡히는지 확인
- 아침 먹었는지 묻는 질문이 1일차 시시콜콜한 질문으로 잡히는지 확인
- 정치/종교 관련 메시지가 1주차 금지어로 잡히는지 확인
- 답장 재촉이 1주차 위험 주제로 잡히는지 확인
- 전 애인 질문이 2주차 위험 주제로 잡히는지 확인

### 상태 테스트

- 메시지 전송 후 즉시 답장이 표시되지 않는지 확인
- `전송됨` -> `읽음` -> `입력 중...` -> `답장 도착` 순서가 유지되는지 확인
- 20회 왕복 후 단계 결과 화면으로 이동하는지 확인
- 경고 3회 누적 시 실패 화면으로 이동하는지 확인

### 데모 테스트

- 3분 안에 오프닝, 캐릭터 선택, 성공 루트, 실패 루트를 모두 보여줄 수 있는지 확인
- 네트워크/API 실패 시 fallback 답변으로 데모가 끊기지 않는지 확인
